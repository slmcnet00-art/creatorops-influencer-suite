# CreatorOps Influencer Suite

AI 기반 인플루언서 발굴, 섭외, 콘텐츠 성과 추적, 보고서 다운로드까지 한 번에 운영하기 위한 로컬 MVP입니다.

## 핵심 기능

- 멀티 브랜드 워크스페이스: 여러 브랜드를 등록하고 브랜드별 캠페인, 추천, 메시지, 성과를 분리 관리
- 브랜드 조건 설정: 브랜드명, 제품, 타깃 페르소나, 포함/제외 키워드, 플랫폼, 카테고리, 팔로워/단가 조건
- AI 후보 추천: 설정값에 맞춰 인플루언서 후보를 점수화하고 추천 이유, 리스크, 페르소나를 기록
- 자동/수동 섭외 분류: AI 추천 기반 섭외는 자동, 직접 작성 섭외는 수동으로 구분
- 메시지 검토함: 제안 메시지 초안을 저장하고 승인 대기, 발송 완료, 응답, 섭외 완료 상태로 관리
- 섭외 완료 풀: 섭외 완료된 인플루언서를 별도 풀에 저장해 재섭외/캠페인 배정에 활용
- 콘텐츠 추적: 인플루언서가 올린 콘텐츠 URL, 조회수, 좋아요, 댓글, 공유, 저장, 전환 수 기록
- 보고서 다운로드: 캠페인 CSV, 성과 CSV, HTML 보고서, 전체 워크스페이스 JSON 백업 지원
- 데이터 소스 원장: 후보별 팔로워, 평균 조회, 참여율, 단가의 출처/신뢰도/갱신 방식을 표시
- 로컬 저장: 브라우저 `localStorage`에 데이터가 저장되어 새로고침 후에도 유지

## 실행 방법

```bash
npm install
npm run dev
```

브라우저에서 아래 주소를 엽니다.

```text
http://localhost:5173
```

## 운영 흐름

1. 좌측 `Brand Workspace`에서 운영할 브랜드를 선택하거나 새 브랜드를 추가합니다.
2. 선택한 브랜드의 조건 설정에서 제품과 타깃 페르소나를 입력합니다.
3. `AI 매칭 실행`을 눌러 해당 브랜드 기준 추천 후보와 추천 이유를 생성합니다.
4. 추천 후보에서 `메시지 검토함`을 눌러 자동 제안 메시지를 검토함에 저장합니다.
5. 직접 작성이 필요하면 크리에이터 상세에서 `제안 보내기`로 수동 제안 메시지를 만듭니다.
6. `제안/응답 발송`에서 `발송 완료`, `응답 처리`, `섭외 완료 저장` 순서로 상태를 관리합니다.
7. 섭외 완료된 인플루언서는 브랜드별 `섭외 완료 풀`에 저장됩니다.
8. 콘텐츠가 업로드되면 `콘텐츠 추적 등록`으로 URL과 성과 데이터를 기록합니다.
9. `추적 데이터 갱신`으로 성과 데이터를 업데이트하고, `성과 보고서 다운로드`로 보고서를 받습니다.

## 현재 범위와 다음 단계

현재 버전은 로컬 MVP입니다. 실제 Instagram, YouTube, TikTok 검색 및 DM 발송은 각 플랫폼 API, 이메일/메시징 계정, 백엔드 서버 연동이 필요합니다. 데이터 구축 방향은 [DATA_ARCHITECTURE.md](./DATA_ARCHITECTURE.md)에 정리했습니다.

다음 단계 후보:

- OpenAI API 기반 실제 후보 리서치 에이전트 연결
- YouTube Data API와 공개 웹/미디어킷 수집 서버 연동
- Instagram/TikTok은 크리에이터 인증 데이터와 허용된 공개 소스 중심으로 수집
- 이메일, Slack, 카카오 채널 등 발송 채널 연동
- GitHub 또는 클라우드 DB에 워크스페이스 동기화
- 팀 계정, 권한, 감사 로그 추가
