# CreatorOps Influencer Suite

브랜드에 맞는 인플루언서와 틱톡 공동구매 셀러를 발굴, 섭외, 협업 운영해 조회수와 전환 KPI가 나오는 콘텐츠를 만들기 위한 로컬 MVP입니다.

## 핵심 기능

- 멀티 브랜드/팀 워크스페이스: 여러 브랜드를 등록하고 브랜드별 캠페인, 추천, 메시지, 성과를 팀 단위로 분리 관리
- 브랜드 조건 설정: 브랜드명, 제품, 타깃 페르소나, 포함/제외 키워드, 플랫폼, 카테고리, 팔로워/단가 조건
- AI 브리프 조건 세팅: 제품, 후킹포인트, 희망 인플루언서 조건을 붙여넣으면 실제 검색에 쓸 브랜드 조건, 발굴 필터, 캠페인 초안, 학습자료를 자동 생성
- 실제 웹 발굴: YouTube Data API와 Google Programmable Search를 연결해 실제 채널/프로필 URL을 후보 DB에 저장
- PM 우선순위/실행 보드: 팀 공유 DB, KPI 구조화, 실제 후보 수집, 발송/응답 로그의 우선순위와 준비도를 대시보드에서 확인
- 브랜드/제품 학습자료: 제품 브리프, 상세페이지 문구, 금지표현, 기존 성과를 엑셀 `.xlsx`, CSV/TSV, Google Sheets 붙여넣기/공개 URL로 등록해 추천과 메시지에 반영
- 캠페인 브랜드 가이드: 캠페인 생성 시 인플루언서 브랜드 가이드 양식을 다운로드하고 작성 파일을 첨부해 캠페인과 학습자료에 연결
- AI 후보 추천: 데이터룸 raw/metric으로 후보를 먼저 점수화하고, API 서버의 OpenAI 보강으로 추천 이유, 리스크, 페르소나, 제안 메시지를 기록
- 광고주 전달용 리스트: 추천/발굴/섭외 완료 후보를 숏폼 리스트업 엑셀 형식으로 다운로드
- KPI 기반 캠페인: 목표 조회수, 전환, 공동구매 주문, 목표 매출, 셀러 섭외 목표를 캠페인별로 관리하고 달성률 확인
- 틱톡 공동구매 셀러 대량 섭외: 조건에 맞는 TikTok 후보를 메시지 검토함에 일괄 저장
- 자동/수동 섭외 분류: AI 추천 기반 섭외는 자동, 직접 작성 섭외는 수동으로 구분
- 연락 채널 라우팅: 이메일 자동 발송 후보와 Instagram/TikTok DM 수동 보조 후보를 분리해 메시지 검토함에서 관리
- 플랫폼 보완 기능 반영: 시장의 검증된 운영 방식을 참고해 캠페인 타입, 미션, 리워드, 검수 플로우를 실제 관리 기능으로 제공
- 메시지 검토함: 제안 메시지 초안을 저장하고 승인 대기, 발송 완료, 응답, 섭외 완료 상태로 관리
- 섭외 완료 풀: 섭외 완료된 인플루언서를 별도 풀에 저장해 재섭외/캠페인 배정에 활용
- 배송/수동 정산 관리: 수취인, 주소, 계좌, 지급 예정액, 운송장, 지급 상태를 기록하고 엑셀/시트로 내보내기
- 콘텐츠 추적: 인플루언서가 올린 콘텐츠 URL, 조회수, 좋아요, 댓글, 공유, 저장, 전환 수 기록
- 보고서 다운로드: 캠페인 CSV, 성과 CSV, HTML 보고서, 전체 워크스페이스 JSON 백업 지원
- 데이터 소스 원장: 후보별 팔로워, 평균 조회, 참여율, 단가의 출처/신뢰도/갱신 방식을 표시
- 로컬 저장: 현재 MVP는 브라우저 `localStorage`에 저장되며, 운영 버전은 팀 공유 DB와 로그인 권한으로 전환 필요

## 운영 흐름

1. 좌측 `Brand Workspace`에서 운영할 브랜드를 선택하거나 새 브랜드를 추가합니다.
2. 선택한 브랜드의 조건 설정에서 제품과 타깃 페르소나를 입력합니다.
3. 이미 작성한 캠페인 브리프가 있으면 `브리프 붙여넣기 + 조건 세팅`에 붙여넣고 `조건 세팅`을 눌러 제품, 후킹포인트, 희망 인플루언서 조건을 검색 조건으로 반영합니다.
4. 브랜드/제품 학습자료가 있으면 엑셀 파일을 업로드하거나 Google Sheets 범위를 붙여넣습니다.
5. 발굴 화면의 `실제 웹 발굴`에서 YouTube API Key 또는 Google Search Key/CX를 연결하고 `실제 검색`을 눌러 실제 후보를 가져옵니다.
6. `AI 매칭 실행`을 눌러 실제 후보 DB 기준 추천 후보를 생성합니다. API 서버에 `OPENAI_API_KEY`가 있으면 추천 이유와 제안 메시지가 보강되고, 없으면 데이터룸 점수화 추천으로 계속 진행됩니다.
7. 캠페인 생성 시 인플루언서 브랜드 가이드 양식을 다운로드하거나 작성된 가이드를 첨부하고, KPI 목표, 목표 조회수, 목표 전환, 목표 주문, 목표 매출, 셀러 섭외 목표를 함께 입력합니다.
8. 추천 후보에서 `메시지 검토함`을 눌러 자동 제안 메시지를 검토함에 저장합니다.
9. 광고주에게 후보를 공유할 때는 AI 추천 후보, 크리에이터 발굴, 섭외 완료 풀의 `광고주용` 버튼으로 숏폼 리스트업 엑셀을 다운로드합니다.
10. TikTok 공동구매 캠페인은 `틱톡 셀러 대량 저장`으로 후보를 일괄 저장합니다.
11. 직접 작성이 필요하면 크리에이터 상세에서 `제안 보내기`로 수동 제안 메시지를 만듭니다.
12. `제안/응답 발송`에서 이메일, Instagram DM, TikTok DM 등 권장 연락 채널을 확인하고 메시지를 복사하거나 프로필을 엽니다.
13. 발송 후 `발송 완료`, `응답 처리`, `섭외 완료 저장` 순서로 상태를 관리합니다.
14. 섭외 완료된 인플루언서는 브랜드별 `섭외 완료 풀`에 저장됩니다.
15. 콘텐츠가 업로드되면 `콘텐츠 추적 등록`으로 URL과 성과 데이터를 기록합니다.
16. `추적 데이터 갱신`으로 성과 데이터를 업데이트하고, `성과 보고서 다운로드`로 보고서를 받습니다.

## 현재 범위와 다음 단계

현재 버전은 로컬 MVP입니다. 한 명의 브라우저에서 기능을 검토하는 용도이며, 실제 제품은 팀원이 로그인해서 같은 워크스페이스를 함께 보는 공유 DB/인증 구조가 필요합니다. 예시 후보는 기본 발굴 리스트에서 숨기고, 실제 후보는 API 연결 후 가져옵니다. YouTube는 공식 API로 구독자/평균 조회까지 저장하고, Instagram/TikTok은 Google 검색 결과의 실제 프로필 URL을 저장한 뒤 팔로워/조회수는 검증 대기로 둡니다. AI 추천은 데이터룸 기준 점수화가 기본이며, `OPENAI_API_KEY`가 설정된 API 서버에서는 `/ai/recommendations/enrich`로 추천 사유와 메시지를 보강합니다. 대량 연락은 공개 협업 이메일 기반 자동 발송을 우선하고, Instagram/TikTok DM은 승인형 또는 수동 보조 채널로 운영합니다. 자동 송금/자동정산은 1차 범위에서 제외하고, 지급 예정액과 지급 상태를 수동으로 기록하는 방식으로 운영합니다. 데이터 구축 방향은 [DATA_ARCHITECTURE.md](./DATA_ARCHITECTURE.md)에 정리했습니다.

다음 단계 후보:

- OpenAI 추천 보강 산출물을 Supabase `ai_generation_runs`/`llm_generation_artifacts`로 저장
- 틱톡 공동구매 셀러 대량 발굴 파이프라인 고도화
- YouTube Data API와 공개 웹/미디어킷 수집 서버 연동
- Instagram/TikTok은 크리에이터 인증 데이터와 허용된 공개 소스 중심으로 수집
- Gmail/Outlook 기반 이메일 발송과 Instagram/TikTok DM 수동 보조 플로우 연동
- Supabase 또는 백엔드 DB에 워크스페이스 동기화
- 팀 계정, 초대, 역할 권한, 클라이언트 보기 권한, 감사 로그 추가

## Production wiring

프론트는 이제 Supabase 공유 저장소와 별도 API 프록시를 옵션으로 지원합니다.

1. `.env.example`을 기준으로 `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`, `VITE_WORKSPACE_ID`를 설정합니다.
2. Supabase SQL Editor에서 `supabase/schema.sql`을 실행합니다.
3. API 키를 브라우저에 직접 두지 않으려면 `VITE_CREATOROPS_API_BASE_URL`을 설정하고 `API_INTEGRATION_CONTRACT.md`의 엔드포인트를 백엔드에 구현합니다.
4. 설정 화면의 `운영 연결 상태`에서 Supabase 저장 여부와 API 프록시 연결 여부를 확인합니다.
